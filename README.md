# GlowCRM — Telegram CRM-бот для запису на сеанси макіяжу

Повноцінна CRM-система, що **повністю працює всередині Telegram-бота** на бібліотеці
[**aiogram 3**](https://docs.aiogram.dev). Клієнти записуються на сеанси прямо в боті, а
**адміністратори отримують власну адмін-панель теж у боті** — без окремого сайту чи застосунку.

## Огляд

GlowCRM — це готове рішення для невеликої б'юті-студії або приватного майстра. Замість сайту
з реєстраціями вся взаємодія відбувається у звичному Telegram:

- **звичайний користувач** бачить дружнє меню із записом на сеанс, своїми бронюваннями та налаштуваннями;
- **адміністратор** (один або кілька) бачить додаткову кнопку та повноцінну панель керування —
  замовлення, статистика, послуги, графік, розсилки, клієнти й керування адмінами.

Усі кнопки — інтерактивні інлайн-клавіатури з кольоровими піктограмами-емодзі. Дані зберігаються
у локальній SQLite-базі (режим WAL), тож бот і панель працюють поверх однієї бази без конфліктів.

> Скріншоти інтерфейсу можна додати у каталог `docs/img/` і підключити сюди.

## Можливості

### Для клієнтів
- **Запис на сеанс** у три кроки: послуга → дата (на місяць уперед, лише робочі дні) → вільний час.
- **Мої записи** — перегляд активних та минулих записів, скасування.
- **Послуги та ціни** — актуальний прайс із тривалістю й описом.
- **Налаштування** — телефон і керування сповіщеннями.
- **Сповіщення** про підтвердження, виконання чи скасування запису.

### Для адміністраторів (панель у боті)
- **Замовлення** — вкладки за статусами (нові / підтверджені / виконані / скасовані), дії:
  підтвердити, виконати, скасувати, видалити; клієнту автоматично надходить сповіщення.
- **Статистика** — нові/підтверджені/виконані записи, записи на сьогодні, кількість клієнтів,
  дохід за виконаними, текстовий графік завантаженості за тиждень.
- **Клієнти** — база з історією записів кожного клієнта.
- **Послуги** — додавання (майстер у кілька кроків), приховування/відновлення, видалення.
- **Графік** — робочі дні тижня, години роботи, крок слоту, горизонт запису.
- **Розсилка** — масове повідомлення всім клієнтам зі ввімкненими сповіщеннями.
- **Привітання** — редагування вітального тексту (підтримує підстановку назви студії).
- **Адміністратори** — додавання/видалення адмінів за username прямо в боті.

Логіка вільних слотів генерується автоматично з урахуванням графіка та вже зайнятих записів —
подвійне бронювання неможливе.

## Технології

| Шар | Технологія |
|-----|------------|
| Telegram-бот | [aiogram 3](https://docs.aiogram.dev) (async, FSM, роутери) |

| База даних | SQLite (режим WAL) |
| Конфігурація | python-dotenv (`.env`) |
| Мова | Python 3.10+ |

## Швидкий старт

```bash
# 1. Клонувати репозиторій
git clone <URL-вашого-репозиторію> glow-crm
cd glow-crm

# 2. Створити віртуальне середовище
python -m venv .venv
# Windows
.\.venv\Scripts\activate
# Linux / macOS
source .venv/bin/activate

# 3. Встановити залежності
pip install -r requirements.txt

# 4. Налаштувати конфіг
copy .env.example .env      # Windows
cp .env.example .env        # Linux / macOS
# відкрийте .env і впишіть BOT_TOKEN та ADMIN_USERNAMES

# 5. Запустити
python run.py
```

У Telegram відкрийте свого бота та надішліть `/start`. Адміністратори побачать кнопку
**Адмін-панель** і зможуть викликати її командою `/admin`.

> Детальніше — у [docs/CONFIGURATION.md](docs/CONFIGURATION.md) та [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).

## Конфігурація

Усі параметри задаються у файлі `.env` (шаблон — `.env.example`):

| Змінна | Обов'язкова | Опис |
|--------|:-----------:|------|
| `BOT_TOKEN` | так | Токен бота від [@BotFather](https://t.me/BotFather) |
| `ADMIN_USERNAMES` | так* | Username адмінів без `@`, через кому (один або багато) |
| `ADMIN_IDS` | ні | Числові Telegram ID адмінів, через кому |
| `BUSINESS_NAME` | ні | Назва студії, що показується клієнтам |

\* Достатньо вказати або `ADMIN_USERNAMES`, або `ADMIN_IDS`. Адмінів можна також додавати
з самої панелі. Повний опис — у [docs/CONFIGURATION.md](docs/CONFIGURATION.md).

## Структура проєкту

```
glow-crm/
├─ run.py                  # Точка входу: python run.py
├─ requirements.txt        # Залежності
├─ .env.example            # Шаблон конфігурації (місце для токена)
├─ README.md
├─ LICENSE
├─ CHANGELOG.md
├─ CONTRIBUTING.md
├─ Dockerfile
├─ docker-compose.yml
├─ docs/                   # Повна документація
│  ├─ ARCHITECTURE.md
│  ├─ CONFIGURATION.md
│  ├─ USER_GUIDE.md
│  ├─ ADMIN_GUIDE.md
│  └─ DEPLOYMENT.md
└─ app/
   ├─ config.py            # Читання .env, список адмінів
   ├─ database.py          # SQLite + SQLAlchemy (WAL)
   ├─ models.py            # User, Service, Booking, Broadcast, Setting
   ├─ seed.py              # Початкові послуги та налаштування
   ├─ services.py          # Бізнес-логіка: слоти, записи, статистика, адміни
   └─ bot/
      ├─ main.py           # Запуск aiogram (Bot, Dispatcher, polling)
      ├─ keyboards.py      # Інлайн-клавіатури (кольорові кнопки)
      ├─ states.py         # FSM-стани діалогів
      ├─ filters.py        # Фільтр доступу IsAdmin
      ├─ utils.py          # Сповіщення та форматування
      └─ handlers/
         ├─ user.py        # Клієнтські сценарії
         └─ admin.py       # Адмін-панель
```

База даних створюється автоматично у `data/glowcrm.sqlite3` при першому запуску.

## Розгортання

Локально достатньо `python run.py`. Для постійної роботи — systemd-сервіс або Docker.
Готові `Dockerfile` та `docker-compose.yml` додані до репозиторію. Деталі —
у [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).

```bash
docker compose up -d --build
```
