# Внесок у проєкт

Дякуємо за інтерес до GlowCRM! Цей документ описує, як долучитися до розробки.

## Налаштування середовища

```bash
git clone <URL>
cd glow-crm
python -m venv .venv
source .venv/bin/activate        # Windows: .\.venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env             # впишіть тестовий BOT_TOKEN
```

## Робочий процес

1. Створіть гілку від `main`: `git checkout -b feat/коротка-назва`.
2. Вносьте зміни невеликими логічними комітами.
3. Перевірте, що проєкт запускається: `python run.py`.
4. Відкрийте Pull Request з описом змін.

## Гілки та коміти

- Назви гілок: `feat/...`, `fix/...`, `docs/...`, `refactor/...`.
- Повідомлення комітів — у наказовому способі, наприклад:
  - `feat: додати нагадування за день до сеансу`
  - `fix: коректно обробляти зайнятий слот`
  - `docs: оновити інструкцію адміна`

## Стиль коду

- Python, дотримуйтесь PEP 8.
- Коментарі та текст для користувачів — українською.
- Не додавайте надлишкових коментарів, що дублюють очевидний код.
- Бізнес-логіку розміщуйте в `app/services.py`, а не в хендлерах.
- Нові тексти/кнопки — у `app/bot/keyboards.py` та відповідних хендлерах.

## Структура для нових функцій

| Що додаєте | Куди |
|------------|------|
| Нову модель/поле | `app/models.py` (+ за потреби `app/seed.py`) |
| Нову бізнес-логіку | `app/services.py` |
| Нову клавіатуру | `app/bot/keyboards.py` |
| Новий клієнтський сценарій | `app/bot/handlers/user.py` |
| Нову адмін-функцію | `app/bot/handlers/admin.py` |
| FSM-стан | `app/bot/states.py` |

## Повідомлення про помилки

Створюючи issue, додайте:
- кроки відтворення;
- очікувану та фактичну поведінку;
- версію Python та ОС;
- релевантні рядки логів (без токена!).

> Ніколи не публікуйте вміст файлу `.env` або токен бота.
