# Інструкція адміністратора

Адмін-панель GlowCRM вбудована в того ж бота. Доступ отримують користувачі, чий username
або Telegram ID вказано в конфігурації (див. [CONFIGURATION.md](CONFIGURATION.md)).

## Вхід до панелі

1. Переконайтеся, що ваш `username` є в `ADMIN_USERNAMES` (або ID у `ADMIN_IDS`).
2. Надішліть боту `/start` — у меню з'явиться кнопка **Адмін-панель**.
3. Або одразу викличте команду `/admin`.

## Розділи панелі

### Замовлення
- Вкладки за статусами: **Нові**, **Підтверджені**, **Виконані**, **Скасовані**.
- Натисніть на замовлення, щоб побачити деталі: послуга, дата, час, клієнт, телефон, коментар.
- Доступні дії:
  - **Підтвердити** (для нових) — клієнт отримає сповіщення про підтвердження;
  - **Виконано** (для підтверджених) — позначити сеанс як проведений;
  - **Скасувати** — скасувати запис і сповістити клієнта;
  - **Видалити** — повністю прибрати запис із бази.
- Списки гортаються кнопками сторінок.

### Статистика
Зведення по студії:
- кількість нових, підтверджених та виконаних записів;
- записи на сьогодні;
- кількість клієнтів;
- дохід за виконаними записами;
- текстовий графік завантаженості за останні 7 днів.

### Клієнти
- Перелік усіх клієнтів із кількістю записів.
- Перегляд картки клієнта: username, ID, телефон, статус сповіщень та історія останніх записів.

### Послуги
- Перегляд усіх послуг (зелений кружок — активна, білий — прихована).
- **Додати послугу** — майстер у 4 кроки: назва → ціна → тривалість → емодзі.
- У картці послуги: увімкнути/вимкнути показ клієнтам або видалити.

### Графік
- Перемикання робочих днів тижня (натисканням).
- Зміна часу початку та завершення роботи (формат `HH:MM`).
- Зміна кроку слоту (у хвилинах) та горизонту запису (днів уперед).
- Зміни миттєво впливають на доступні клієнтам слоти.

### Розсилка
- Введіть текст — його отримають усі клієнти зі ввімкненими сповіщеннями.
- Після завершення показується статистика доставки.

### Привітання
- Редагування вітального повідомлення, яке бачить клієнт на `/start`.
- Можна використати підстановку `{business}` — підставиться назва студії.

### Адміністратори
- Перегляд незмінних адмінів із `.env` та доданих через панель.
- **Додати адміна** — за username (без `@`).
- Видалення доданих через панель адмінів кнопкою.

## Сценарій типового дня

1. `/admin` → **Замовлення → Нові**.
2. Відкрити кожен новий запис, **Підтвердити** або **Скасувати**.
3. Після проведення сеансу — позначити **Виконано**.
4. За потреби — **Розсилка** з акцією чи нагадуванням.
5. Періодично переглядати **Статистику** для контролю завантаженості.

## Поради

- Тримайте перелік **Послуг** актуальним — приховуйте тимчасово недоступні, а не видаляйте.
- Налаштуйте **Графік** під реальні години роботи, щоб клієнти не бачили зайвих слотів.
- Кілька майстрів можуть бути адмінами одночасно — додайте всіх у `ADMIN_USERNAMES`.
